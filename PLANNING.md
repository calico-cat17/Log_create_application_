## 확정 탭 구조

```
홈 / 모임 / 후기 / 마이  (4개 탭)
```

### 공통 레이아웃
- 상단 헤더: _log 로고(파란색 자물쇠) + 알림 버튼 (패널 드롭다운)
- 홈 헤더: _log 로고 / 나머지 탭: 탭명 텍스트
- 하단 탭 바: 홈 / 모임 / 후기 / 마이

| 섹션 | 내용 | 운영진 추가 |
|------|------|------------|

| 번개 목록 | 좌우 스냅 캐러셀 (인접 카드 peek) | — |

## 탭 2 — 모임 (정기모임 / 번개모임 세그먼트)

### 정기모임

**상태별 버튼 흐름**

| 상태 | 표시 |
|------|------|
| D-7 이상 (신청 열림) | "뒷풀이만 신청" + "정기모임 신청" 2버튼 |
| 미오픈 | 회색 disabled 버튼 |
| 테마 미확정 (카드 클릭) | 스낵 바텀시트: "아직 테마 확정 전입니다" + 취소/희망테마 신청하기 |
| 테마 확정 + 배정 완료 | 방켓팅 버튼 + 뒷풀이 버튼 |
| 방켓팅 완료 | 비활성 버튼 + 뒷풀이 버튼 |
| 운영진 | 카드 우상단 배정하기 버튼 → 자동배정 모달 + 확정하기 |

**정기모임 상세**
- 기본: 제목·날짜·위치
- 테마 확정 전: 확정 전 메시지 + 희망테마 신청하기 버튼
- 테마 확정 후: 테마표(테마명·가격·모이는 시간·시작 시간) + 배정표 (표 형태)
- 끝난 모임: 상세 + 배정 결과 + 테마표 + 사진

**정기모임 신청 흐름**
- 방켓팅 버튼 클릭 → 1·2·3지망 드롭다운 (테마 선택 시 장르·가격 즉시 표시)
- 신청 → "신청을 확정하시겠습니까?" 최종 확인 알림창

**뒷풀이**
- 장소·최대 인원 표시
- "뒷풀이에 참여하시겠습니까? 뒷풀이만 참여는 출석으로 인정되지 않습니다." 안내 문구

### 번개모임

| 구분 | 내용 |
|------|------|
| 다가오는 번개 | 신청·모집 인원·상세·FAB 번개올리기 |
| 필터 (다가오는) | 방탈출/비방탈출, 모집마감 안보기 |
| 끝난 번개 | 상세·사진 |
| 필터 (끝난) | 내 모임만 보기, 방탈출/비방탈출 |
| 번개 등록 | 우하단 FAB 버튼만 (상단 버튼 없음) |

## 탭 3 — 후기

| 섹션 | 내용 |
|------|------|
| 방탈 랭킹 | 테마명·리뷰 수·평균 평점, 더보기 → 전체 랭킹 (장르 표시) |
| 전체 후기 | 별점순/최신순/공포도순/난이도순 정렬 |

> 후기 작성 버튼 없음 (모임 탭 끝난 모임에서만 작성)

## 목 데이터 구조 (prototype.js 기준)

```js
THEMES      // 테마 ID → { name, genre, price }
MEMBERS     // 부원 배열 → { name, gen, att, warn, mile }
MEETINGS    // 모임 배열 → { id, title, type, area, date, gen, themes[], attendees, afterParty }
RANKINGS    // 랭킹 배열 → { rank, icon, name, reviews, cleared, rating }
APPLICANTS  // 신청자 배열 → { name, p1, p2, p3 }
```
